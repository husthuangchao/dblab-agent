How it works

Smart agent, dumb tools

The model does all the reasoning and planning. The tools just execute SQL and return rows. The agent is instructed — and equipped — to prove every behavioural claim.

Ask

“Does MySQL support window functions?” or compare a behaviour across all three engines.

Probe

The agent writes the smallest SQL that proves it and runs it on the real connection.

Read

Actual columns and rows come back — including errors — and feed the model.

Answer

A grounded reply, labelling tested facts apart from background knowledge.

Features

Built to be trusted

🧪

Evidence, not vibes

Six pure SQL tools (list, describe, exec, batch). No tool ever calls the model — all judgement stays in the LLM.

🔭

See every step

The chat streams each tool call and its result. Expand any step to inspect the exact SQL and rows.

🖼️

Read your screenshots

Drop in a SQL-error screenshot; the multimodal model (Zhipu GLM-4V) reads it and diagnoses.

🔌

Bring any model

OpenAI-compatible. DeepSeek by default; OpenAI, Qwen, or a local Ollama by changing two lines.

🐳

One command

docker compose up brings the agent and all three databases online, pre-seeded with demo data.

🔐

Your keys, encrypted

Manage text + vision keys from the admin page; stored Fernet-encrypted and gitignored.

Quickstart

Up and running in one command

Spins up the agent plus MySQL 8, PostgreSQL 17, and openGauss 3.0 — seeded and ready.

# clone, add your LLM key, launch
git clone https://github.com/husthuangchao/dblab-agent.git
cd dblab-agent
cp .env.example .env        # put your OpenAI-compatible key in .env
docker compose up -d        # MySQL 8 + PostgreSQL 17 + openGauss 3.0, pre-seeded

# then open  http://localhost:9527
